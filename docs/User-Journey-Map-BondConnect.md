# User Journey Map — BondConnect

---

## Persona 1: Mike

Mike is a 38-year-old who has worked in commercial real estate for 20 years. He's married, has 3 children, and his kids play soccer. On the weekends he catches up with work.

---

## Journey 1: Starting point

1. Mike downloads the app.
2. Creates an account.
3. The app may prompt for **notification and calendar permissions** (optional).
4. He wants to connect with his high school friend Tom, whom he hasn't seen in years.
5. He opens the **Calendar** or **Events** tab (calendar view with [+ Add]), taps **[+ Add]**, and chooses from a list of three options — **Quick Phone Call** (15 min), **Deep Talk** (30 min), or **Event**. He picks **Deep Talk**.
6. He selects the person to speak to — Tom (already registered in the system).
7. The system shows a review screen; everything looks ok. He accepts.
8. The system sends an invitation to Tom, and the event is in **Pending** status.

---

## Journey 2: Scheduling with multiple people

1. Opens the app (already installed and logged in).
2. The main home screen opens.
3. User opens the **Calendar** or **Events** tab (calendar view with [+ Add]), taps **[+ Add]**, and chooses **Event** from the three options.
4. User enters the length of the event (1 hour to any length), description, time and date, location, RSVP optional.
5. User selects the people to invite (e.g. Tom, Ashley, Jamie, Adam — as many as they want). Contacts must already be registered in the system.
6. The system shows a review screen; everything looks ok. User accepts.
7. The system sends an invitation to the selected contacts, and the event is in **Pending** status.

---

## Journey 3: Adding a new contact

1. Open the app (already installed and logged in).
2. The main home screen opens.
3. User clicks on the **Contacts** tab and **[+ Add]**, which shows:
   - Name, phone/email
   - Status (Friend, Partner, Family, Work)
   - City
   - Cadence (e.g. how often to connect: Weekly, Bi-weekly, Monthly)
4. The system shows a review screen; everything looks ok. User accepts.
5. The system sends an invitation to the new contact, and the contact is in **Pending** status.

---

## Journey 4: Notification

1. Opens the app (already installed and logged in).
2. The main home screen opens.
3. A notification appears; user can click it (optional), which leads to the **Inbox** tab.
4. User clicks on the **Inbox** tab, which shows:
   - Number of requests
   - Request sent by: contact name, status (Friend, Partner, Family, Work), location/city, time last sent, duration
5. User **accepts**, **reschedules**, or **declines**. The system shows a confirm/review step as needed. User ends up in the **Contacts** tab. (Optional: can go to the **Calendar** or **Events** tab.)

---

## Journey 5: Not able to do the dates

1. Opens the app (already installed and logged in).
2. The main home screen opens.
3. Notification appears; user can click it (optional), which leads to the **Inbox** tab.
4. User clicks on the **Inbox** tab and sees the reason for the request:
   - Sent by: contact name, status, location/city, time last sent
   - Duration, description, and length of the event
5. User can **Accept**, **Reschedule**, or **Decline**.
   - **Accept** — System shows a confirm screen; everything looks ok. User ends up in the Contacts tab.
   - **Reschedule** — System shows suggested times (e.g. three slots) or user picks a new date/time. User sends a new time proposal to the sender.
   - **Decline** — Sender gets a notification that it doesn’t fit the schedule.
6. The system sends to the user, and the event remains in **Pending** status.

---

## Journey 6: New contact — invite from friend, partner, family, or colleague

1. User receives a standard text message, email, or social message that contains a link.
2. New user clicks the link. **If the app is already installed**, the link opens the app (Splash: “You’re invited to connect”). **If not**, the link goes to the app store to download the app.
3. Creates an account.
4. New user gets a tutorial (short video on how to use the app) or can **skip** it.
   - **Skip** — Notifications and a **Support** tab help users find videos and troubleshooting.
   - **Continue** — Tutorial videos cover: Social Bar, scheduling, Contacts, Inbox, and Home.
5. The app may prompt for **notification and calendar permissions** (optional).
6. User has all tabs ready to use.

---

## Journey 7: All the tabs

1. Opens the app (already installed and logged in).
2. The main home screen shows:
   - **Home**
   - **Contacts**
   - **Calendar**
   - **Chat or Messenger**
   - **Events**

### Home tab

- Social bar meter
- Weekly capacity bar
- Suggested / recommended contacts (auto request)
- Status of connections throughout the month

### Contacts tab

- All contact names and locations
- Social bar meter level per contact and their status
- Cadence per contact (e.g. Weekly, Bi-weekly, Monthly)
- Add contact

### Calendar tab

- Calendar with all dates of the month
- Upcoming events (click to see details for that day)
- Add Event

### Chat or Messenger tab

- All text messages by person
- Status / number of contacts for that day (e.g. “Contacts today: 3”)
- Add Event

### Events tab

- Opens the same calendar view as the Calendar tab, with [+ Add] to create a Quick Call, Deep Talk, or Event. Event creation includes: contact(s), description, length, date/time, location, RSVP (optional).

---

## Journey 8: Post-Event Check-in

1. After an event or call (e.g. Deep Talk with Tom), the app prompts the user.
2. User is asked how the connection felt: **Energizing**, **Neutral**, or **Draining**.
3. User can optionally say when they want to reconnect again: **Soon**, **Later**, or **Pause**, and add a note.
4. This feedback closes the loop and helps the Social Bar and suggestions.
